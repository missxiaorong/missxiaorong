HTML5页面移动设备自适应宽度总结(2016.1.6)

下一篇文章：

只要在页面head标签加入

<head> <meta name="viewport" content="width=device-width,initial-scale=1.0,minimum-scale=0.5, maximum-scale=2.0, user-scalable=yes"/> <meta name="apple-mobile-web-app-capable" content="yes"/> <meta name="format-detection" content="telephone=no" > </head> viewport是网页默认的高度和宽度
第一行：
width=device-width： 表示宽度是设备屏幕的宽度
initial-scale=1.0： 表示初始的缩放比例
minimum-scale=0.5： 表示最小的缩放比例
maximum-scale=2.0： 表示最大的缩放比例
user-scalable=yes：表示用户是否可以调整缩放比例
第二行：
设定iphone端页面全屏
第三行：
取消数字被识别为电话号码

如果是想要一打开页面，自动以原始比例显示，且不允许用户进行修改，则是：
<meta name="viewport" content="width=device-width,initial-scale=1.0,minimum-scale=1.0, maximum-scale=2.0,user-scale=no"> 一、网页的布局不使用绝对宽度，也不能使用绝对宽度的元素。
例如：css不能指定像素宽度width:100px;只能指定百分比宽度 width:10%，或者width:auto ;
二、字体大小使用相当大小的字体
body{font:normal 100% Helvetica, Arial, sans-serif;}这句的代码指定，字体的大小是页面默认的大小的100%，即16px
h1{font-size:1.5em;} h1大小是默认字体大小的1.5倍 ，即24px;
small{font-size:0.875em;}small元素的字体是默认字体的0.875倍，即14px

在这里顺便总结下em和rem的使用及区别
1、em是css2引进来的。使用em时，需要注意:
页面默认的字体大小是16px,转换为10px，将会减少一定的计算。
body{font-size:62.5%; /*10 ÷ 16 × 100% = 62.5% */ } 10 ÷ 16 × 100% = 62.5% 使用"1em=10px"改变默认的"1em=16px"
h1{font-size:2.4em; /*2.4em × 10 = 24px */ }
small{font-size:1.4em; /*1.4em × 10 = 14px */}

在使用em做单位时一定要知道父元素的的设置，因为"em"是一个相对值，是一个相对于父元素的值,"em"的值是不固定的。
计算公式是:

em = 需要转换的像素值 / 父元素的font-size的值

2、rem(root em ,根em )是css3新增的一个相对单位。

用rem为元素设定大小时，仍然是相对大小，但相对的只是html根元素，所以它既可以做到只修改根元素就成比地调整所有字体的大小，又可以避免字体 大小逐层变化导致的复合混乱效果。

body{font-size:62.5%; /*10 ÷ 16 × 100% = 62.5% */ } 10 ÷ 16 × 100% = 62.5% 使用"1em=10px"改变默认的"1em=16px"
h1{font-size:2.4rem; /*2.4rem × 10 = 24px */ }
small{font-size:1.4rem; /*1.4rem × 10 = 14px */}

计算公式是:

rem = 需要转换的像素值 /html根元素的font-size的值（即body中设定的值）

三、使用流动布局，即各个区块的位置是浮动的，不是固定不变的。
.main{float:left;width:60%;} .rightBar{float:right;width:30%;}
如果页面宽度太小放不下两个元素，后面的元素会自动滚动到前面元素的下方，而不会在水平方向溢出，避免了水平滚动条的出现。

四、图片的自适应
img{max-width:100%;}

五、使用媒体查询
@media screen and (max-width:960px){ body{ background:#000; } }
以上是media的一个标准写法，代码意思是，当页面大小小于960px时，执行下面的css.
screen的意思是告知设备在打印页面时使用衬线字体，在屏幕上显示时用无衬线字体。如果网站不需要考虑用户去打印时可以，可以直接写
@media screen and (max-width:960px){ body{ background:#000; } }
还有以下几种写法：
@media (min-width:800px) and (max-width:1200px) { ... }
@media (min-width:800px) and (max-width:1200px) and (orientation:portrait) { ... }
意思为媒体查询仅在宽度为 800 到 1200 像素且方向是纵向时才能激活。如果其中一个条件为 False，则无法应用媒体查询规则。
@media (min-width:800px) or (orientation:portrait) { ... }
如果宽度至少是 800 像素或方向是纵向的，则会应用该规则。
@media (not min-width:800px) { ... }

附加：
如果目标是横向模式智能手机，则使用： @media (min-width: 321px) { ... }
如果目标是纵向模式智能手机，则使用： @media (max-width: 320px) { ... }
如果目标是横向模式 Apple iPad，则使用： @media (orientation: landscape) { ... }
如果目标是纵向模式 iPad，则使用： @media (orientation: portrait) { ... }
iPad 上使用的是 orientation 媒体特性，而 width 用于 Apple iPhone 之上。
主要是因为 iPhone 不支持 orientation 媒体特性。
注意，由于IE8即不支持HTML5也不支持css3 media,所以在使用media时，需要加载两个js文件，来保证代码实现兼容效果：
!--[if IE;9] <script src=https://oss.maxcdn.com/libs/html5shiv/3.7.0/html5shiv.js"></script> <script src="https://oss.maxcdn.com/libs/respond.js/1.3.0/respond.min.js"></script> ![endif] 以下代码可以让IE以最高的文档模式展现效果： <meta http-equiv="x-ua-compatible" content="IE=edge"，chrome="1">
